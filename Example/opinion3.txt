


---

1. 현재 문제점

현재 생성 루트:

[한국어 인풋 + 외모태그] 
→ 그대로 Image API에 전달 
→ 즉시 이미지 생성

문제:

Danbooru 태그 조합 생성 단계가 없음

Image API로 한국어 프롬프트가 그대로 전달됨

“깡캐의 셀카” 같은 입력이 영어 태그 변환 없이 바로 사용됨

또한, 사용자 인풋이 아닌 ai출력이더라도 sns/메세지 각 이미지 프롬프트가 주입되지 않는 문제



--

2. 수정되어야 할 정상 생성 순서

반드시 아래 순서를 강제할 것:

1) AI가 사용자 인풋(한국어) + SNS/메세지(경우에 따라) 각 이미지 프롬프트 수신
2) 상황에 맞는 Danbooru 영어 태그 조합 생성
3) 생성된 태그 조합을 Image API 프롬프트로 같이 주입
4) Image API 호출
5) 이미지 생성

단계별 명확 정의

STEP 1 — 입력 수집

입력값:

사용자 텍스트 (한국어)

상황 컨텍스트 (SNS용, 메세지용 등)

캐릭터 기본 외모 태그


이 단계에서는 절대 Image API 호출 금지


---

STEP 2 — 태그 변환 전용 AI 처리

AI 역할:

한국어 입력을 해석

상황에 맞는 Danbooru 형식 영어 태그 조합 생성

결과는 반드시 영어 태그만 출력

문장형 출력 금지


예시:

입력:

여성 user의 셀카

출력 예시:

1girl, selfie, looking_at_viewer, phone_in_hand, casual_smile, indoor, upper_body


---

STEP 3 — Image API 주입 구조

Image API에는 아래만 전달:

[기본 외모 태그] + [AI가 생성한 영어 Danbooru 태그 조합]

절대 한국어 원문을 포함하지 말 것.


---

3. 구조 강제 조건

한국어 원문은 절대 Image API에 직접 전달 금지

태그 생성이 완료되지 않으면 이미지 생성 차단

태그 생성 실패 시 이미지 생성 금지

생성 단계는 비동기라도 순서 고정

4. 토글형 퀵 액세스 UI 구조 정의

UI 구조 요구사항

형식: 2단계 토글 구조

[아이콘 버튼]
   ↓ 클릭
[퀵 액세스 리스트 펼침]
   ↓ 항목 클릭
[패널창 오픈]

QUICK REPLY와 동일한 UX 구조.

구성 요소

1. Floating Icon Button (고정 위치)


2. Expandable Quick List (세로 리스트)


3. 각 항목 클릭 시 Side Panel 또는 해당 Modal 바로 열림
(예시- 폰 아이콘을 클릭. 각 퀵액세스 아이콘 및 본 패널 여는 qr같은 창이나옴. 만약 내가 이모티콘 아이콘을 선택한다면 이모티콘 모달 창이 바로열림. 전화 아이콘을 누르면 바로 전화걸림. 퀵센드 아이콘 누르면 기능 바로 작동 등)
